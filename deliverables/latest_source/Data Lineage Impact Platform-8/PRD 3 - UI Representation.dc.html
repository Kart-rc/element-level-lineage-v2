UI & Representation

How thousands of services, datasets and jobs become legible: an altitude-based navigation model and four coordinated lenses over one graph — lineage, interactions, impact, and provenance.

Executive summary

You can’t draw a 2,000-node graph and expect anyone to read it. Throughline’s UI is built on one principle: never render the whole graph — navigate by altitude. Start at domains, drill to the team-aligned business application, then to services, datasets, and columns, loading only what’s in view. Four lenses share this navigation so a user keeps their place while switching between “where does data flow,” “who calls whom,” “what breaks,” and “how do we know.” A working prototype already demonstrates the model.
