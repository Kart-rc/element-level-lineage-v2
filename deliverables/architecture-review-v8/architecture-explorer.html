Entity & edge schema — what v2 adds over v8

  

    
Entity
stable entityId (ULID) · URN as name, not join key · sameAs aliases · env axis · lifecycle state · (urn, schemaVersion) in a separate version registry

    
Edge (v8 fields kept)
level · channel · transform · path {guard, codeRef, frequency} · signals[] · lastObservedAt — v8's distinctive path-qualified design is retained verbatim

    
Edge (new in v2)
validFrom/validTo bitemporal versions · lifecycle · conflict flag · edge_observations provenance table with idempotency key · integer score + band (never decimals)

    
Governance objects
snapshots (logical watermarks) · gate_decisions (pinned triple) · waivers (expiring, approver-bound) · policies (Rego versions, ramp stage) · tombstones · calibration log

The OTel correction (GAP-D5)
  Raw spans at ~50 rps/service would be ~ billion spans/day at this slider position — three to four orders of magnitude above everything else. The edge-aggregating collector folds spans into per-edge observations (hot 5-min · warm hourly · cold on-fire ⇒ 36.6 obs/edge/day), bounding OTel load to interaction edges × 36.6 — independent of request rate. Still ~7.6× the OpenLineage stream. v8 counted zero of it.
