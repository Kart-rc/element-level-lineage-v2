§1 · Lineage collection

Seven mechanisms, one gateway, zero mutual trust

Three mechanisms produce the baseline (what the code should do), three produce runtime truth (what actually happened), one produces authoritative types. None trusts another — the fusion engine reconciles them.

Baseline — what code should do
static + llm

Static + LLM analyzer

Repo scan per PR + nightly. sqlglot AST column lineage for SQL/dbt; pinned LLM (temp 0) for DataFrame chains & dynamic SQL. Golden-set gated: precision ≥90% or LLM-only edges cap at Inferred. Content-hash cache.

declared

Lineage contracts (lineage.yaml)

The escape hatch for uncovered systems (stored procs, vendor ETL). Owner-attributed, schema-validated, 180-day TTL. Capped at Probable 75 until runtime-corroborated.

ci diff

CI diff service

Re-derives lineage for changed paths on every PR; keeps the baseline current and feeds the gate (§4).

Runtime — what actually happened
spark · dask · airflow

OpenLineage listeners

RunEvents with column-lineage facets on every job run. The signals that promote edges from Inferred to Verified.

otel

OTel edge-aggregating collector

Raw spans never leave the collector tier: folded into per-edge observations, flushed hot/5min · warm/hourly · cold/on-fire. 732k obs/day at 2k — vs ~8.6 billion raw spans. The GAP-D5 correction.

Type truth
registry

Schema-registry sync

Confluent SR subjects, warehouse INFORMATION_SCHEMA snapshots, repo OpenAPI/protobuf. Registry wins all type conflicts; registry-vs-runtime drift becomes an owner finding, never silently reconciled.

§2 · Fusion engine

From "an event arrived" to "the graph changed" — one idempotent pipeline

Stages 4–7 are where six signals become one trustworthy edge: deterministic identity, per-attribute conflict rules, calibrated confidence. Every write is append-only — any past graph state is reconstructable.

1
GatewayauthN · schema validation · metadata-only enforcement (reject → DLQ, SLI-alerted)
2
Raw archiveKafka → Iceberg, append-only. The replay/backfill source: fusion bugs are repairable by shadow rebuild + atomic snapshot swap.
3
Normalizewire format → assertion; URN construction (ADR-005): urn:tl:{env}:{type}:{authority}:{path}#{fragment}
4
Identity resolution (ADR-006)exact URN → alias registry → normalization rules → human-confirmed suggestions. Never probabilistic auto-merge — a wrong merge silently poisons every downstream impact answer.
5
Observation upsertidempotency key (source, runId, eventId) — retries can never double-count into confidence; event-time merge — late events never overwrite newer state.
6
Conflict resolution (ADR-007)per-attribute precedence: existence runtime>declared>static>llm · transform static>llm · types: registry wins · ownership never LLM-only. Cold-path rule: asserted-but-unobserved edges stay visible, always Possible in impact.
7
Confidence scoring (ADR-008)agreement × recency; baseline-only cap 64; declared cap 75; conflict −10. Bands (Verified ≥85 · Probable 65–84 · Inferred <65) move by quarterly calibration; model version pinned in every snapshot.
8
Lifecycle (ADR-009)active → stale (stream 7d · batch 35d) → retired (tombstones: OL delete, CI file deletion, registry delete, attestation). "Gone" and "quiet" are different facts.
9
Append-only graph write (ADR-004)close old version, open new — never UPDATE. Snapshots are logical watermarks; AS OF queries reproduce any past state.
10
CDC fan-outPostgreSQL logical replication → CSR traversal projection (22 MB @2k) · search index · roll-ups · cache invalidation.

Worked example — one edge's life orders.events#total_amount → orders_raw#total_amount

T0 · baseline build

Born static + LLM INFERRED 61

Analyzer parses the Kafka producer + Avro schema; LLM maps order.totalAmount() → event.total_amount. Guard order.status=='CONFIRMED', codeRef OrderService.java:88, frequency never-observed. Baseline-only cap: can never gate.

T1 · Spark run confirms

Runtime-corroborated VERIFIED 96

OpenLineage COMPLETE with column facet joins the same entityIds via the deterministic identity chain. Frequency hot, rowsOut 124,903. Both observations retained in edge_observations — nothing overwritten.

T2 · signals disagree

Registry vs stale static CONFLICT −10

Registry says decimal(12,2); a stale static assertion says int. Matrix: registry wins the type; edge flagged conflict; triage event. Disagreement is surfaced, never averaged away.

T3 · pipeline deleted

Explicit retirement RETIRED

CI diff sees the landing job's file deleted → tombstone. Edge leaves impact traversal; queryable 13 months as "was connected until 2027-01". Under v8's decay-only model this was indistinguishable from a quiet quarter-end job.

§4 · PR flow (CI/CD gate)

Webhook → manifest → diff → impact → policy → status check, in 30 seconds

The gate's authority is earned mechanically: observe → warn → block per domain, a waiver workflow with real approvers and expiry, and a circuit-breaker that demotes any rule crying wolf. On platform failure the gate fails open with a warn annotation — Throughline being down must never mean the org cannot merge.

1
Webhook (~1s)PR opened/updated
2
Manifest resolution (~2s)throughline.yaml maps changed paths → service URNs. Monorepo-safe; unmapped paths → visible warn, never a silent skip.
3
Lineage re-derivation (~5s)sqlglot + LLM cache on changed files → schema deltas and codeRef intersections (logic-only changes — the GAP-C1 fix).
4
Impact on pinned snapshot (~10s)§3 flow; decision is reproducible and appealable forever.
5
OPA policy (~2s)Rego packages per domain, stewards own them via PR; ramp stage is a policy input; Possible tier can never block (v8 FR-I6 retained).
6
Status check + comment (~10s slack)impact table, coverage line, deep link to blast radius. p95 budget 30s end-to-end · timeout 120s → fail-open + warn.

setTotalAmount(int cents) → setTotalAmount(BigDecimal), plus a new currency field.

  1. Manifest maps services/orders/** → svc:…orders-svc; diff finds total_amount int→decimal changed, currency added.
  2. Impact: 3 Confirmed warnings + 1 Possible review + 1 Possible breaking (partner export, cold path).
  3. Domain revenue is in warn ramp stage → OPA: WARN. Nothing Confirmed is breaking, and Possible can never block.
  4. Non-blocking status check + comment: impact table, "3 of 4 paths runtime-confirmed", deep link. Developer merges with eyes open; the partner-export owner is notified.

normalize_orders.sql: o.total_amount as net_amount → as net_revenue

  1. Diff: fct_orders — net_amount removed, net_revenue added (sqlglot sees the rename in the AST).
  2. Impact on pinned snapshot: churn_features#avg_order_value reads the old name — Confirmed breaking (hot, Dask-verified; rename × join-transform = break).
  3. Domain is in block stage → OPA: BLOCK with rule id, owning team (ml-platform), and two exits: coordinate the rename in the same change, or obtain a waiver.
  4. Waiver rules: approver must be the impacted owner or domain steward (never the PR author), expiry ≤90 days, fully audited.
  5. Counterfactual: had the PR changed the formula for net_amount with no rename, v8's schema-diff-only gate stays silent; here the codeRef intersection emits the warn-tier transform-logic-changed notice.
The trust circuit-breaker For each policy rule: waiver_rate_30d = waivers / block_decisions. Above 10%, the rule auto-demotes block → warn and pages its owner. v8's "false-positive budget" becomes a mechanism instead of a slide-word — the gate cannot silently burn credibility.
§5 · How the four flows compose

Three shared spines, one closed loop

§1 Collection
7 mechanisms → gateway → Kafka → Iceberg
→
§2 Fusion
identity → conflict → confidence → lifecycle → append-only graph + snapshots
→
§3 Impact · §4 Gate
CSR projection → reports & decisions (pinned snapshot + policy + model)

…and the loop closes: every merge updates the baseline (CI diff → collection), every gate decision and waiver feeds calibration and triage (fusion), and the graph the next PR is judged against is already current.

Spine 1 — Identity

Every mechanism joins through the same deterministic URN grammar and stable entityId. Collection without shared identity is seven disagreeing inventories.

Spine 2 — Snapshots

Fusion writes append-only; impact and the gate read pinned watermarks. One discipline makes every blocking decision reproducible and appealable.

Spine 3 — Confidence with teeth

Conflict matrix + calibration make the fused graph more trustworthy than any single signal. Possible never blocks; cold paths always surface — honest in both directions.

■ Verified ≥85 — gate-eligible ■ Probable 65–84 — surfaced, cautioned ▨ Inferred <65 — dashed, never gates ■ Breaking ■ Warning ■ Review
